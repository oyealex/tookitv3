在实验性工作流程中归档已完成的变更。

**输入**：可选择在 `/opsx:archive` 后指定变更名称（例如 `/opsx:archive add-auth`）。如果省略，检查是否可以从对话上下文中推断。如果模糊或不确定，你必须提示选择可用变更。

**步骤**

1. **如果未提供变更名称，提示选择**

   运行 `openspec list --json` 获取可用变更。使用 **AskUserQuestion 工具** 让用户选择。

   只显示活动变更（尚未归档的）。
   如果可用，包含每个变更使用的模式。

   **重要**：不要猜测或自动选择变更。始终让用户选择。

2. **检查制品完成状态**

   运行 `openspec status --change "<name>" --json` 检查制品完成状态。

   解析 JSON 以了解：
   - `schemaName`：使用的工作流程
   - `artifacts`：制品及其状态（`done` 或其他）的列表

   **如果任何制品不是 `done`：**
   - 显示未完成制品列表的警告
   - 提示用户确认继续
   - 如果用户确认则继续

3. **检查任务完成状态**

   读取任务文件（通常为 `tasks.md`）以检查未完成的任务。

   计算带有 `- [ ]`（未完成）vs `- [x]`（完成）的任务。

   **如果发现未完成任务：**
   - 显示未完成任务数量的警告
   - 提示用户确认继续
   - 如果用户确认则继续

   **如果没有任务文件：** 在没有任务相关警告的情况下继续。

4. **评估增量规范同步状态**

   检查 `openspec/changes/<name>/specs/` 处是否存在增量规范。如果不存在，在没有同步提示的情况下继续。

   **如果存在增量规范：**
   - 将每个增量规范与 `openspec/specs/<capability>/spec.md` 处相应的主规范进行比较
   - 确定将应用哪些更改（添加、修改、删除、重命名）
   - 在提示之前显示组合摘要

   **提示选项：**
   - 如果需要更改："立即同步（推荐）"、"不同步就归档"
   - 如果已经同步："立即归档"、"仍然同步"、"取消"

   如果用户选择同步，使用 Task 工具（subagent_type: "general-purpose"，提示："使用 Skill 工具调用 openspec-sync-specs 进行变更 '<name>'。增量规范分析：<包含分析的增量规范摘要>">）。无论选择如何，都继续归档。

5. **执行归档**

   如果不存在则创建归档目录：
   ```bash
   mkdir -p openspec/changes/archive
   ```

   使用当前日期生成目标名称：`YYYY-MM-DD-<change-name>`

   **检查目标是否已存在：**
   - 如果存在：失败并出错，建议重命名现有归档或使用不同日期
   - 如果不存在：将变更目录移至归档

   ```bash
   mv openspec/changes/<name> openspec/changes/archive/YYYY-MM-DD-<name>
   ```

6. **显示摘要**

   显示归档完成摘要，包括：
   - 变更名称
   - 使用的模式
   - 归档位置
   - 规范同步状态（已同步/跳过同步/无增量规范）
   - 任何警告的说明（未完成的制品/任务）

**成功时的输出**

```
## 归档完成

**变更：** <change-name>
**模式：** <schema-name>
**归档到：** openspec/changes/archive/YYYY-MM-DD-<name>/
**规范：** ✓ 已同步到主规范

所有制品完成。所有任务完成。
```

**成功时的输出（无增量规范）**

```
## 归档完成

**变更：** <change-name>
**模式：** <schema-name>
**归档到：** openspec/changes/archive/YYYY-MM-DD-<name>/
**规范：** 无增量规范

所有制品完成。所有任务完成。
```

**成功时的输出（带警告）**

```
## 归档完成（带警告）

**变更：** <change-name>
**模式：** <schema-name>
**归档到：** openspec/changes/archive/YYYY-MM-DD-<name>/
**规范：** 跳过同步（用户选择跳过）

**警告：**
- 已归档 2 个未完成的制品
- 已归档 3 个未完成的任务
- 增量规范同步被跳过（用户选择跳过）

如果这不是故意的，请检查归档。
```

**错误时的输出（归档已存在）**

```
## 归档失败

**变更：** <change-name>
**目标：** openspec/changes/archive/YYYY-MM-DD-<name>/

目标归档目录已存在。

**选项：**
1. 重命名现有归档
2. 如果是重复的，删除现有归档
3. 等到不同日期再归档
```

**护栏**
- 如果未提供变更名称，始终提示选择
- 使用制品图（openspec status --json）进行完成检查
- 不要因警告阻止归档——只是通知和确认
- 移动到归档时保留 .openspec.yaml（它随目录移动）
- 清晰显示发生的事情摘要
- 如果请求同步，使用 Skill 工具调用 `openspec-sync-specs`（代理驱动）
- 如果存在增量规范，始终运行同步评估并在提示之前显示组合摘要
